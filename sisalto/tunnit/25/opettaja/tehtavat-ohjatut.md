# Ohjatut tehtävät — oppitunti 25

## Aktiviteetti 1 — Missä ihminen tarvitaan? (15 min)

Luokittele tilanteet:

- tavallisen vastaanottokuittauksen lähettäminen
- hinnanalennuksen hyväksyminen
- tietokannan rivin poistaminen
- lisätiedon pyytäminen epäselvään viestiin
- lääkinnälliseen tai oikeudelliseen päätökseen vaikuttava suositus

Kysy jokaisesta: mikä on virheen vaikutus, voidaanko toiminto perua ja kuka kantaa vastuun?

## Aktiviteetti 2 — Rakenna päätösnäkymä (15 min)

Hyväksyntäportissa pitää näkyä:

- mitä päätöstä pyydetään
- mihin aineistoon ehdotus perustuu
- agentin lyhyt havaittava perustelu
- riskitaso ja seuraukset
- hyväksy, muokkaa, hylkää ja eskaloi -vaihtoehdot

## Aktiviteetti 3 — Mitä tapahtuu hiljaisuudessa? (15 min)

Ryhmät määrittävät yhdelle rahaa koskevalle päätökselle:

- vastausajan
- varahenkilön
- eskalointiketjun
- turvallisen oletustoiminnon

Kriittistä päätöstä ei hyväksytä automaattisesti pelkän vastaamattomuuden vuoksi.

## Aktiviteetti 4 — Palautteesta hallituksi muutokseksi (20 min)

Täytä ketju:

| Vaihe | Kuvaus |
| --- | --- |
| Lokitettava päätös |  |
| Ihmisen perustelu |  |
| Toistuva ongelma, jota etsitään |  |
| Muutos, jota ehdotetaan |  |
| Kuka hyväksyy muutoksen? |  |
| Millä testillä vaikutus tarkistetaan? |  |

Korosta, ettei päätöksen tallentaminen vielä muuta kielimallia tai agentin toimintaa.

## Opettajan tarkistuslista

- Portit sijaitsevat riskikohdissa, eivät kaikkialla.
- Päätöksentekijä saa tarpeellisen taustatiedon.
- Aikarajan jälkeinen toiminta on määritelty.
- Palaute muuttuu toiminnaksi vain arvioidun kehitysprosessin kautta.
