# Opiskelutehtävät — Valinnainen tuotantolaboratorio

> Valitse yksi reitti: kuva, ääni tai musiikki, video tai koodi. Reitistä riippumatta osoitat saman taidon: johdat käyttötarpeesta kriteerit, arvioit ensimmäistä versiota niiden avulla, muutat yhtä asiaa ja vertaat muutoksen vaikutusta ennen julkaisu- tai hylkäyspäätöstä.

Jos sinulla ei ole sopivaa palvelua, käytä opettajan antamia kahta esimerkkituotosta ja tee arviointi- sekä korjaussuunnitelma ilman tilin luomista.

## Vaihe 1 — Määritä käyttötarkoitus

Kuvaa ensin tilanne, jossa tuotosta todella tarvitaan. Kirjoita näkyviin:

- mitä tuotat
- kenelle tuotos on tarkoitettu
- missä sitä käytetään
- mikä on tärkein onnistumisen ehto
- mitä tuotoksessa ei saa olla.

Näin arviointisi perustuu tarpeeseen eikä siihen, sattuuko ensimmäinen versio näyttämään kiinnostavalta.

**Läpi kulkeva esimerkki:** Kirjasto tarvitsee verkkosivulleen 20 sekunnin äänettömän videon, joka näyttää kirjan palauttamisen kolmessa vaiheessa. Ennen tuottamista sovitaan neljä kriteeriä: vaiheet näkyvät oikeassa järjestyksessä, jokaisen tekstityksen ehtii lukea rauhassa, esineet ja tila säilyvät johdonmukaisina eikä kuvassa ole tunnistettavia asiakkaita.

## Vaihe 2 — Valitse reittisi ja kirjoita syöte

### A. Kuva

Kerro syötteessä käyttötarkoitus, pääkohde, sommittelu, visuaalinen ilme ja kuvasuhde. Rajaa pois vain ne asiat, jotka vaarantaisivat käyttötarkoituksen tai tekisivät kuvasta harhaanjohtavan.

Arviointikriteereiksi sopivat esimerkiksi käyttötarkoitus, sommittelu, yksityiskohtien oikeellisuus, harhaanjohtamattomuus ja saavutettavuus.

### B. Ääni tai musiikki

Kerro syötteessä käyttötarkoitus, kesto ja rakenne. Kuvaa tempo, äänimaailma ja mahdollinen puhe havaittavina ominaisuuksina sekä rajaa pois käyttöön sopimattomat elementit.

Arviointikriteereiksi sopivat esimerkiksi rakenteen toteutuminen, puheen ymmärrettävyys, äänen tasapaino, tekniset häiriöt ja käyttöoikeus.

### C. Video

Kirjoita 2–4 kohtauksen kuvakäsikirjoitus. Kuvaa kesto, tapahtumat, kamera, liike, ääni tai tekstitys ja lopussa välittyvä ydinasia. Kirjaston esimerkissä jokainen palauttamisen vaihe saa oman kohtauksensa, jotta järjestystä ja lukuaikaa voidaan arvioida erikseen.

Arviointikriteereiksi sopivat esimerkiksi jatkuvuus, liikkeen uskottavuus, viestin selkeys, äänen ja kuvan yhteys sekä saavutettavuus.

### D. Koodi

Kirjoita ensin vaatimus: mitä ohjelma tekee, millaisen syötteen se saa ja millaisen tuloksen se palauttaa. Nimeä ympäristö, riippuvuudet ja tärkeät virhetilanteet sekä vähintään kolme testiä. Käytä vain turvallista harjoitusympäristöä.

Arviointikriteereiksi sopivat esimerkiksi toimivuus, luettavuus, virheiden käsittely, testien läpäisy ja se, ettei koodi tee pyytämättömiä toimintoja.

## Vaihe 3 — Tuota versio 1 ja arvioi

Tallenna syöte ja ensimmäinen tuotos muuttamattomina. Arvioi tuotosta 3–5 kriteerillä, jotka päätit ennen tuottamista, ja täytä taulukko:

| Kriteeri | Havainto versiosta 1 | Korjaustarve |
| --- | --- | --- |
|  |  |  |
|  |  |  |
|  |  |  |

Kirjaston videon versiossa 1 vaiheet ovat oikeassa järjestyksessä, ja esineet sekä tila säilyvät johdonmukaisina. Toinen tekstitys vaihtuu kuitenkin niin nopeasti, ettei vertaisarvioija ehdi lukea sitä. Tämä on havaittava puute suhteessa ennalta sovittuun kriteeriin.

## Vaihe 4 — Muuta yhtä asiaa

Nimeä yksi ongelma ja tee sitä vastaava muutos syötteeseen tai toteutukseen. Älä vaihda samalla työkalua, tyyliä ja kaikkia asetuksia, sillä silloin et voi tietää, mikä muutoksista vaikutti tulokseen.

> Muutan ____, koska versiossa 1 havaitsin ____.

Tuota versio 2. Kirjaston esimerkissä syötteeseen lisätään vain yksi vaatimus: toinen kohtaus kestää seitsemän sekuntia. Muu sisältö ja muut asetukset pidetään ennallaan.

## Vaihe 5 — Vertaa ennen ja jälkeen

| Kriteeri | Versio 1 | Versio 2 | Mitä muutos osoitti? |
| --- | --- | --- | --- |
|  |  |  |  |
|  |  |  |  |
|  |  |  |  |

Kirjoita johtopäätös. Kerro, paransiko nimetty muutos tuotosta, mistä vaikutus näkyy ja mitä et voi vielä päätellä yhden kokeen perusteella. Kirjaston videon versiossa 2 tekstitys ehditään lukea ja muut kriteerit täyttyvät edelleen. Vertailu tukee päätelmää, että pidempi toinen kohtaus korjasi juuri nimetyn puutteen, mutta se ei vielä osoita menetelmän toimivan kaikissa videoissa.

## Vaihe 6 — Vastuullisuustarkistus ja päätös

Käy ennen julkaisemista tai käyttöönottoa läpi seuraavat kysymykset:

- Oliko syöte ja lähdeaineisto luvallista käyttää?
- Saako tuotoksen julkaista suunnitellussa käytössä?
- Mitä pitää vielä tarkistaa ihmisenä?
- Voiko tuotos johtaa jotakuta harhaan tai aiheuttaa vahinkoa?
- Miten kerrot tekoälyn osuudesta?

Tee tarkistuksen perusteella selvä päätös: **julkaise, hylkää tai palauta korjattavaksi**. Kirjaston esimerkkivideo voidaan julkaista, kun version 2 tekstitys on osoitettu luettavaksi, lähdeaineiston käyttö on luvallista, kuvassa ei ole tunnistettavia henkilöitä ja ihminen on tarkistanut ohjeen oikeellisuuden. Jos yksikin porttiehto jää avoimeksi, versiota ei julkaista vielä.

**Tallennettava tuotos:** laboratorioloki, joka sisältää käyttötarkoituksen, syötteen, versiot 1 ja 2, kriteeritaulukot, yhden nimetyn muutoksen, johtopäätöksen ja vastuullisuustarkistuksen.
