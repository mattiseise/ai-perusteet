# Opiskelutehtävä — Pohja, muokkaus ja tarkistus

## Tehtävä 13.1 — Tee yksi käyttöohje työparina

**Tavoite:** Käytät tekoälyä rajatusti luonnoksen tekemiseen ja kriittiseen lukemiseen. Sinä päätät tehtävän, korjaat sisällön ja hyväksyt lopputuloksen.

Tehtävän aikana samasta ohjeesta syntyy useita näkyviä vaiheita. Älä korvaa aiempaa versiota uudella, sillä juuri luonnoksen, omien korjaustesi ja testilukijan palautteen välinen ero osoittaa, missä oma harkintasi vaikutti.

### 1. Rajaa todellinen ohje

Valitse arjesta, opiskelusta tai työstä pieni tehtävä, jonka tunnet itse riittävän hyvin. Ohjeessa saa olla enintään kahdeksan vaihetta. Kirjoita ennen tekoälyn käyttöä:

- kohderyhmä
- tavoiteltu lopputulos
- kolme asiaa, joiden pitää olla ohjeessa oikein
- lähde tai oma havainto, josta tarkistat nämä asiat.

Valitse aihe, jossa pystyt erottamaan sujuvan arvauksen oikeasta toimintatavasta. Jos kirjoitat esimerkiksi laitteen tai verkkopalvelun käyttöohjetta, tarkista painikkeiden nimet todellisesta näkymästä. Jos kuvaat yhteisön käytäntöä, käytä voimassa olevaa ohjetta tai omaa todennettavaa havaintoasi.

Älä käytä aineistona henkilötietoja tai luottamuksellista sisältöä.

### 2. Pyydä luonnos

Pyydä käytössäsi olevaa hyväksyttyä tekoälypalvelua tekemään luonnos. Kerro kohderyhmä, tavoite, enimmäispituus ja haluttu rakenne. Pyydä mallia merkitsemään epävarmat yksityiskohdat sen sijaan, että se arvaisi niitä.

Säilytä luonnos muuttamattomana versiona 1.

Versio 1 on työn lähtöjälki. Sen tarkoitus ei ole onnistua täydellisesti, vaan näyttää, mitä tekoäly ehdotti ennen kuin oma tietosi ja päätöksesi muuttivat tekstiä.

### 3. Tee oma muokkaus

Käy luonnos kohta kohdalta läpi. Merkitse vähintään kolme omaa päätöstä:

| Luonnoksen kohta | Päätös: säilytä, muokkaa tai poista | Perustelu ja tarkistuslähde |
| --- | --- | --- |
|  |  |  |
|  |  |  |
|  |  |  |

Tee korjattu versio 2. Pelkkä kieliasun siistiminen ei riitä, jos sisältö, järjestys tai rajaus vaatii muutosta.

Kirjaa päätökset niin, että ulkopuolinen ymmärtää niiden syyn. ”Muokkasin tätä paremmaksi” on liian väljä perustelu. ”Painikkeen nimeä ei ollut todellisessa käyttöliittymässä, joten korvasin sen tarkistetulla asiointipolulla” yhdistää muutoksen havaintoon.

### 4. Käytä tekoälyä testilukijana

Avaa uusi keskustelu. Anna tekoälylle versio 2, kerro kohderyhmä ja pyydä sitä nimeämään:

- kohta, jossa aloittelija voisi jäädä jumiin
- termi, jota ei ole selitetty
- oletus, jota ohje ei kerro ääneen.

Pyydä palautetta, älä valmista uudelleenkirjoitusta. Arvioi jokainen ehdotus itse: **hyväksy, muokkaa tai hylkää**. Perustele päätös.

Testilukijan palaute kohdistuu versioon 2, ei alkuperäiseen luonnokseen. Näin saat tietoa siitä, mitä omien korjaustesi jälkeen on vielä epäselvää. Muista silti, että testilukijan ehdotus voi olla ristiriidassa lähteen tai käyttötarkoituksen kanssa. Silloin perusteltu hylkäys on oikea ratkaisu.

### 5. Viimeistele ja dokumentoi

Palauta tai tallenna:

- alkuperäinen luonnos
- oma muokattu versio
- testilukijan palaute
- lopullinen ohje
- päätösloki, jossa on vähintään kolme perusteltua valintaa.

Kirjoita loppuun 4–6 virkettä: missä tekoäly nopeutti työtä, missä sen luonnos ei tuntenut todellisuutta ja mikä lopputuloksessa on selvästi sinun ratkaisuasi.

Kokoa pohdinta yhdestä konkreettisesta ketjusta. Nimeä luonnoksen kohta, oma korjauksesi, testilukijan havainto ja lopullinen päätöksesi. Näin pohdinta ei jää yleiseksi arvioksi tekoälyn hyödyistä ja haitoista.

> **Tarkista lopuksi:** Pystyt osoittamaan sekä sen, mitä tekoäly ehdotti, että sen, mitä sinä päätit.
