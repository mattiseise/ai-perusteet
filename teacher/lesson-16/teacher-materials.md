# Lesson 16: Opettajan materiaalit

## Oppimistulokset ja konteksti

**Mitä opiskelija pitää ymmärtää:**
1. Tekoäly voi tuottaa koodia nopeasti ja tehokkaasti
2. Pyyntöjen laatu suoraan vaikuttaa saatujen koodin laatuun
3. Tekoälyn koodi on AINA testattava ennen käyttöä
4. Koodin omaksuminen on yhtä tärkeää kuin sen pyytäminen

## Yleisiä väärinkäsityksiä

| Väärinkäsitys | Oikea ymmärrys |
|---|---|
| "Tekoäly kirjoittaa täydellisen koodin" | Tekoäly tuottaa pohjan — opiskelija testaa ja korjaa |
| "Kopioida koodi on OK, jos se toimii" | Ei ymmärrykse = riskejä + oppimisen puute |
| "Pyyntö ei ole tärkeä, vaikea yksityiskohdista" | Pyyntöjen laatu määrittää tuloksen |
| "Testaus on turhaa, koodi näyttää hyvältä" | Testaus paljastaa virheet, joita silmällä ei näe |

## "Check For Understanding" -kysymykset

- Mikä ero on huonolla ja hyvälla pyynnöllä?
- Mitkä ovat 5 asiaa, jotka pitää lisätä pyyntöön?
- Miksi tekoälyn koodi saattaa näyttää hyvältä mutta silti olla väärä?
- Nimeä kolme reunatapausta, joita sinun pitäisi testata
- Miksi omaksuminen on tärkeää kopioinnin jälkeen?

## Oppitunnin rakenne (90 min)

| Aika | Vaihe |
|------|-------|
| 0–10 min | Johdanto: näytä github copilot tai chat-avustaja |
| 10–35 min | Itsenäinen lukeminen + keskustelu pyyntöjen merkityksestä |
| 35–55 min | Tehtävä 1 — opiskelijat pyytävät koodia ja dokumentoivat (opettaja kiertää) |
| 55–75 min | Tehtävä 2 — vertaus huonoihin vs hyviin pyyntöihin |
| 75–90 min | Yhteenveto + kotitehtävät |

## Opettajan checklist

- [ ] Tekoälypalvelu on käytettävissä (ChatGPT, Claude, GitHub Copilot)
- [ ] Opiskelijoilla on pääsy, tai näytät ruudullasi
- [ ] Dokumentointilomakkeita on tulostettuna (tai digitaalisina)
- [ ] Testausesimerkit ovat valmiina (average(), count_vowels())
- [ ] Opiskelijat tietävät, mitä "reunatapaus" tarkoittaa
