# Opettajamateriaali – Lesson 23

## Oppitunnin ydinajatukset

Suunnittelumallit opettavat agentille, **kuinka ajatella oikein**:
- **ReAct**: joustava, iteratiivinen
- **Ketjuajattelu**: systemaattinen, vaihe vaiheelta
- **Moniagenttijärjestelmät**: jaettu työ, erikoistuminen

---

## ReAct — yksinkertainen analogia

Ihmisen ratkaisuprosessi:
1. Ajattele, mitä tarvitset
2. Tee jotain
3. Katsele, mitä tapahtui
4. Ajattele uudelleen
5. Tee seuraava askel

Agentti tekee samaa — ääneen.

---

## Ketjuajattelu — numeroitu lista

```
1. Ongelma: mitä asiakas kysyy?
2. Rajoitteet: mitä sääntöjä?
3. Valinnat: mitä vaihtoehtoja?
4. Päätös: kumpi valitsemme?
```

Jokainen vaihe riippuu edellisestä. Ei hyppyjä.

---

## Moniagenttijärjestelmät — jaettu vastuu

**Hierarkkinen**:
- Johtaja-agentti näkee kokonaiskuvaa
- Muut agentit ovat spesialisti

**Yhteistyö**:
- Agentit keskustelevat keskenään
- Yhteinen päätöksenteko

---

## Harjoitukset

1. ReAct: Opiskelijat kirjoittavat lokit
2. Ketjuajattelu: Opiskelijat listaa vaiheet
3. Valinta: Opiskelijat päättävät, kumpi malli
4. Moniagentti: Opiskelijat piirtävät kaavion

---

## Yhteinen virheet

1. ReAct ei pääty → Aseta maksimum iteraatiot
2. Ketjuajattelu liian jäykkä → Siihen se on tarkoitettu
3. Moniagentti liian monimutkainen → Aloita yhdellä, lisää tarvittaessa
