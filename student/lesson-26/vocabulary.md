# Lesson 26: Sanasto — Projektipaja

| Termi | Määritelmä |
|-------|-----------|
| **Käyttötapaus** (use case) | Tosielämän scenario, jossa agentin/botin käyttäjä saa hyötyä siitä. Esim. "Agentti vastaa FAQ:hin". |
| **Prototyyppi** (prototype) | Yksinkertainen, ei täydellinen versio projektista, jota voi testata ja parantaa. |
| **MVP** (Minimum Viable Product) | Pienin versio tuotteesta, joka on silti hyödyllinen. Ei tarvitse olla täydellinen. |
| **Feature** (ominaisuus) | Yksi tekemisen asia, jonka agentti/botti osaa tehdä. Esim. "vastaa kysymyksiin" on feature. |
| **Edge case** | Epätavallinen, odottamaton tilanne, johon agentti voi kohdata. Esim. tyhjä input, hyökkäysprompt. |
| **Testaus** (testing) | Prosessi, jossa tarkistetaan, että agentti toimii oikein eri tilanteissa. |
| **Dokumentaatio** (documentation) | Kirjoitettu selitys siitä, mitä agentti tekee, kuinka se toimii, ja kuinka sitä käytetään. |
| **Demo** | Esitys, jossa näytät projektin toiminnassa. Lyhyt, selkeä, ei liian tekninen. |
| **Arkkitehtuuri** (architecture) | Suunnitelma siitä, miten eri osat projektissa liittyvät toisiinsa. |
| **Integraatio** (integration) | Yhteyden luominen agentin/botin ja ulkoisen järjestelmän välille (API, tietokanta, palvelu). |
