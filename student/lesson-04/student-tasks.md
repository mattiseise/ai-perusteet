# Opiskelutehtävät

## Tehtävä 4.1: Konteksti kahteen kertaan — Huono vs. Hyvä — Vertailu

### Tavoite
Ymmärtää käytännöllisesti, kuinka dramaattisesti konteksti muuttaa tekoälyn vastauksen. Kokeile samaa kysymystä ilman kontekstia ja hyvin rakennetulla kontekstilla, ja analysoi eroja.

### Ohjeet (ryhmätehtävä, 2–3 henkilöä)

1. Valitse yhteinen IT-ongelma tai kysymys, joka koskee sinun opiskeluasi. Esimerkiksi: "Kuinka optimoin tietokantahakuja?" tai "Miten debugain Python-ohjelmaa?"

2. **Kerta 1: Ilman kontekstia**
   - Kirjoita tekoälylle vain kysymys ilman mitään kontekstia.
   - Esim.: "Kuinka debugain Python-ohjelmaa?"
   - Kopioi tekoälyn vastaus taulukkoon.

3. **Kerta 2: Hyvällä kontekstilla**
   - Kirjoita sama kysymys, mutta lisää konteksti viiden komponentin mukaan:
     - Rooli: "Olen IT-opiskelija ja aloittelija Python-koodauksessa."
     - Taustatieto: "Olemme kirjoittaneet yksinkertaisen laskinteriaison, joka kuulostaa antavan väärät tulokset."
     - Tavoite: "Tarvitsen apua löytää virhe ja ymmärtää, kuinka debugata systemaattisesti."
     - Rajaukset: "En halua käyttää IDE:n debuggeria, haluan oppia print-pohjaiset metodit."
     - Esimerkki: "Tässä on koodin olennainen osa: `def calculate(a, b): return a * b / 0`"
   - Kopioi tekoälyn vastaus taulukkoon.

4. **Vertailu**
   - Täytä vertailutaulukko:

| Aspekti | Vastaus 1 (ilman kontekstia) | Vastaus 2 (hyvä konteksti) |
|---------|-------------------------------|--------------------------|
| Pituus (sanaväli) | | |
| Tekniikoiden taso | | |
| Mallin relevanssi kysymykseesi | | |
| Käytännöllisenä hyödyllisyys | |
| Spesifisyys | |

5. **Pohdi ja dokumentoi**:
   - Mitkä osat vastauksesta 2 olivat parempia kuin vastauksessa 1?
   - Miksi konteksti muutti vastauksen?
   - Missä kontekstin osassa oli suurin vaikutus?

### Odotettu tuotos
Täytetty taulukko ja 3-4 lauseen pituinen pohdintateksti.

**Jos teet tehtävän yksin:**
Valitse itsellesi sopiva IT-aihe ja suorita kaikki neljä vaihetta. Vertaile näyttöjä.

## Tehtävä 4.2: Kontekstin rakentaminen oikean ongelman pohjalta — Käytännön kirjoitus

### Tavoite
Oppia rakentamaan tehokas konteksti oikean, henkilökohtaisen IT-ongelman ympärille. Harjoitella kontekstin viiden komponentin tunnistamista ja articulointia.

### Ohjeet (ryhmätehtävä, 2–3 henkilöä)

1. Kukin jäsen valitsee todellisen IT-ongelman tai projektin, jota hän käsittelee nyt tai hiljattain käsitteli.
   - Esimerkiksi: "Verkon konfiguroiminen kotiinettä", "Tietokannan suunnitteleminen", "Ohjelman debuggaaminen", "Dokumentaation kirjoittaminen"

2. Dokumentoi ongelmasi **avoimiksi kysymys**:
   - Yksinkertaisuus: Mitä haluat tekoälystä tietää ongelmastasi?

3. Jäsentele kysymys viiden komponentin mukaan:

| Komponentti | Sinun vastaus |
|------------|---|
| **Rooli** | Kuka olet? (Opiskelijataso, aiempi kokemus) |
| **Taustatieto** | Mitä on jo tehty? Mitä tiedät jo ongelmasta? |
| **Tavoite** | Miksi kysyt? Mitä haluat saavuttaa? |
| **Rajaukset** | Mitkä tekniikat/lähestymistavat haluat välttää tai korostaa? |
| **Esimerkki** | Voitko antaa konkrettisen näytteen (koodi, virheilmoitus, kuva)? |

4. Kirjoita kontekstin perusteella **lopullinen, terävä kysymys** tekoälylle:
   - Käytä kaikki viisi komponenttia – yhdessä kappaleessa tai listeissä.
   - Varmista, että se on selvä ja tarkka.

5. **Palaute**:
   - Jaa konteksti-kysymyksesi ryhmäkavereiden kanssa.
   - Antakaa toisillenne palautetta: Onko konteksti selkeä? Puuttuu jotain?

### Odotettu tuotos
- Täytetty taulukko
- Yhden kappaleen (200-300 sanan) konteksti-kysymys
- Hakaan palaute toiselta ryhmän jäseneltä (1-2 lausetta)

**Jos teet tehtävän yksin:**
Täytä taulukko omasta ongelmastasi ja kirjoita lopullinen konteksti-kysymys. Lue se ääneen ja kysy itseltäsi: "Voiko tekoäly ymmärtää, mitä haluan?"
