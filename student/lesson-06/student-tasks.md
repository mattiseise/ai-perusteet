# Opiskelutehtävät

## Tehtävä 6.1: Multimodaalisen kontekstin vertailu — Teksti vs. Kuva

### Tavoite
Ymmärtää käytännössä, kuinka kuvakaappaus muuttaa tekoälyn kykyä auttaa IT-ongelmassa. Nähdä, missä multimodaalisuus auttaa eniten.

### Ohjeet (ryhmätehtävä, 2–3 henkilöä)

1. **Valitsette IT-ongelman**, joka sisältää visuaalisen elementin. Esimerkiksi:
   - Virheilmoitus näytöllä
   - Verkon konfiguraatio-näyttö
   - Tietokanta-hallintaohjelman UI-vika
   - Lokien tulostus päätteessä (näkyy väreillä ja muotoilulla)

2. **Kerätään kaksi versiota kontekstista:**
   - **Versio A (vain teksti):** Kuvaile ongelma sanoin. Kirjoita mitä näet, mitä väriä virhe on, mitä tekstiä näkyy.
   - **Versio B (teksti + kuvakaappaus):** Sama tekstikuvailu PLUS kuvakaappaus ongelmasta.

3. **Lähetetään molemmat tekoälylle** (parhaiten samalle mallille, esim ChatGPT-4V tai Claude 3):
   - Kysyy samaa kysymystä kahdesti — ensin Versiolla A, sitten Versiolla B.
   - Dokumentoikaa molemmat vastaukset.

4. **Vertailu-taulukko:**

| Aspekti | Versio A (vain teksti) | Versio B (teksti + kuva) |
|---------|------------------------|-------------------------|
| Vastak vastauksen spesifisyys | | |
| Tutkittavien ratkaisuiden lukumäärä | | |
| Vastauksessa mainittu visuaalinen tieto | | |
| Kuinka hyödyllinen vastaus | | |
| Token-kustannus (arvio) | | |

5. **Analyysi:**
   - Mikä versio oli parempi?
   - Millä tavoin kuvakaappaus muutti tekoälyn vastauksen?
   - Missä tilanteissa kuvakaappaus olisi olennainen vs. turhaa?

### Odotettu tuotos
- Täytetty vertailu-taulukko
- 2-3 lauseen analyysi: "Kuvakaappaus auttoi silloin kun... Kuvakaappaus ei auttanut silloin kun..."

**Jos teet tehtävän yksin:**
Valitse itsellesi IT-ongelma. Kirjoita tekstikuvaus. Ota kuvakaappaus. Tee molemmat testit ja vertaa.

## Tehtävä 6.2: Multimodaalisen IT-kontekstin rakentaminen — Käytännön projektiverifiointi

### Tavoite
Oppia rakentamaan tehokas multimodaalinen konteksti oikeasta IT-ongelmasta käyttäen tekstiä, kuvakaappauksia ja lokkeja strategisesti.

### Ohjeet (ryhmätehtävä, 2–3 henkilöä)

1. **Valitsette todellisen IT-projektin tai ongelman**, joita käsittelette nyt. Esimerkiksi:
   - Tietokanta on hidas
   - Verkkopalvelu ei vastaa
   - Ohjelman UI:ssa on bugi
   - Palvelin on kaatunut

2. **Kerää konteksti-materiaalia:**
   - **Teksti:** Kirjoita lyhyesti ongelma, rooli, tavoite (oppitunnin 4 malli).
   - **Kuvakaappaus(et):** Ota 1-2 kuvakaappaus, joka näyttää ongelman.
   - **Lokit:** Kopioi relevantit lokin rivit (max 10-20 riviä, älä 500).
   - **Koodi (jos relevanttia):** Näytä kyseinen koodin osa, max 10-20 riviä.

3. **Dokumentoi kukin osa merkinnöillä:**

| Materiaali | Sisältö | Token-arvio | Miksi tärkeä? |
|-----------|---------|---------|---------|
| Teksti | [mitä kirjoitit] | | |
| Kuvakaappaus | [kuvaus] | | |
| Lokit | [mitä rivejä] | | |
| Koodi | [mitä näytät] | | |

4. **Testaa multimodaalinen konteksti:**
   - Kokoa kaikki materiaali yhdeksi kysymykseksi tekoälylle (multimodaalinen malli).
   - Dokumentoi: "Tekoälyn vastaus käytti seuraavaa materiaalia: [mikä]."
   - Pohdi: "Olisiko vastaus parempi tai huonompi ilman kuvakaappauksia / lokkeja / koodia?"

5. **Arvio strategiasta:**
   - Mikä osa multimodaalisesta kontekstista oli arvokkain?
   - Mikä olisi ollut turhaa?
   - Kuinka muuttaisit kontekstia seuraavalla kerralla?

### Odotettu tuotos
- Täytetty materiaalin taulukko
- Yhden sivun raportti:
  - Mikä on projektin/ongelman
  - Miten kasasit multimodaalisen kontekstin
  - Mikä toimi ja mikä ei
  - Oppimasi asiat multimodaalisuudesta

**Jos teet tehtävän yksin:**
Dokumentoi omasi IT-ongelma kerätyllä multimodaalisella kontekstilla. Testaa. Reflektoi.
