# Opiskelutehtävät

## Tehtävä 10.1: Sama tehtävä, kolme työkalua — Testaa ja dokumentoi

### Tavoite
Oppia testaamaan samaa tehtävää usealla eri AI-työkalulla, dokumentoimaan tulokset ja tekemään johtopäätökset siitä, mitä eroja niillä on.

### Ohjeet (ryhmätehtävä, 2–3 henkilöä)

1. **Valitse tehtävä ja työkalut**
   - Tehtävä: Kirjoita Python-funktio, joka tarkistaa, onko sähköpostiosoite voimassa oleva.
   - Tai: Kirjoita lyhyt kuvaus siitä, miten verkon toiminta (IP-osoitteet) toimii.
   - Valitse kolme seuraavista työkaluista:
     - ChatGPT (https://chat.openai.com)
     - Claude (https://claude.ai)
     - Microsoft Copilot (https://copilot.microsoft.com)
     - Muut saatavilla olevat kielimallit

2. **Testaa tehtävä jokaisessa työkalussa**
   - Kirjoita **täsmälleen sama kehotus** (prompt) jokaiseen työkaluun.
   - Kopioi vastaus ja dokumentoi se taulukkoon alla.

3. **Dokumentointi-taulukko**
   - Täytä tämä taulukko:

| Työkalu | Vastauksen pituus | Lähtökohta | Selkeysaste (1-5) | Erityispiirteet |
|---------|------------------|-----------|------------------|----------------|
| Työkalu 1 | [esim. 200 sanaa] | [esim. Alkaa selityksestä] | [1-5] | [esim. Sisältää koodin] |
| Työkalu 2 | | | | |
| Työkalu 3 | | | | |

4. **Analyysi**
   - Mikä työkalu antoi parhaaksi koetussa vastauksen? Miksi?
   - Oliko jokin työkalu terävämpi koodeissa, toinen paremmassa selityksessä?
   - Mitä oppisit siitä, kuinka valita oikea työkalu samanlaiselle tehtävälle tulevaisuudessa?

### Odotettu tuotos

- Täytetty vertailu-taulukko kolmesta työkalusta
- Dokumentoitu vastaukset (voi olla kuvakaappaus tai kopioitu teksti)
- Kirjallinen analyysi (3-5 lausetta) siitä, mitkä olivat erot ja miksi ne ovat ammattilaisesti tärkeät

**Jos teet tehtävän yksin:**
Voit testata kahta työkalua ja dokumentoida niiden väliset erot samalla tavalla.

## Tehtävä 10.2: Kuvan generointi: Testaa ja arvioi

### Tavoite
Oppia käyttämään kuvan generointityökaluja, ymmärtämään niiden vahvuuksia ja rajoituksia, sekä arvioimaan generoidun kuvan laatu ja laillisuus.

### Ohjeet (ryhmätehtävä, 2–3 henkilöä)

1. **Valitse kuvan generaattori**
   - Käytettävissä olevat vaihtoehdot:
     - DALL-E (https://openai.com/dall-e-3)
     - Midjourney (tai vaihtoehtoinen saatavilla oleva työkalu)
     - Stable Diffusion (jos pääsy on saatavilla)

2. **Luo kolme kuvaa**
   - Kuva 1: "Futuristinen IT-toimisto, missä työntekijät tekevät koodausta"
   - Kuva 2: "Abstrakti AI-yhteyksien verkko, kuntaryt värit"
   - Kuva 3: Oma kehotuksesi (esimerkiksi jokin IT-aiheinen kuva)

3. **Dokumentointi: Vertailutaulukko**
   - Kuinka kauan kesti? (sekunteja / minuutteja)
   - Kuinka tarkka oli kuva promptysa verrattuna?
   - Onko siinä jotakin virheellistä tai harhaanjohtavaa?
   - Voitko käyttää tätä kuvaa yrityksen markkinointiin? Miksi / miksi ei?

4. **Analyysi: Oikeudet ja laillisuus**
   - Kuka omistaa generoidun kuvan?
   - Voitko käyttää sitä kaupallisiin tarkoituksiin?
   - Mitä sinun täytyisi tehdä, jos halusit käyttää sitä?

### Odotettu tuotos

- Kolme generoidun kuvaa (kuvakaappaus tai tallennettu tiedosto)
- Dokumentoitu taulukko (aika, tarkkuus, virheet)
- Kirjallinen analyysi oikeuksista (2-3 lausetta)
- Johtopäätös siitä, missä tapauksessa kuvan generaattori olisi hyvä valinta ammatilliselle projektille

**Jos teet tehtävän yksin:**
Voit tehdä yhden tai kaksi kuvaa ja analysoida ne samalla tavalla.

## Tehtävä 10.3: Komentorivin tekoäly­integraatio — Tutustuminen (valinnainen)

### Tavoite
Ymmärtää, että tekoäly on käytettävissä myös komentorivin kautta, ei pelkästään verkkoselaimessa.

### Ohjeet (ryhmä tai yksin, 1–2 henkilöä)

Jos sinulla on pääsy komentoriville ja haluateste offline-tyyvälineitä:

1. **Asenna ollama** (https://ollama.ai) tai käytä Copilot-komentoriviversion.
2. **Lada malli ja testaa yksinkertaista kehotusta:**
   - Esimerkiksi: `ollama run llama2 "Mitä on API?"`
3. **Dokumentoi:**
   - Mitä mallia käytit?
   - Kuinka kauan vastaus kesti?
   - Erosiko vastaus siitä, minkä saisit verkkopohjaisesta ChatGPT:stä?
   - Mikä olisi ammattilaisesti hyödyllistä siinä, että tekoäly on komentorivillä?

### Odotettu tuotos

- Dokumentaatio asennuksesta (tai yritys asennusyritys, jos se epäonnistui)
- Yksinkertainen testi: prompt, vastaus ja havainto
- Johtopäätös: Miksi kehittäjä voisi haluta komentorivin tekoälyä?

**Jos tämä on liian tekninen:**
Voit ohittaa tämän tehtävän ja keskittyä tehtäviin 10.1 ja 10.2.
