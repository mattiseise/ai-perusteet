# Lesson 27: Sanasto — Arviointi ja Reflektio

| Termi | Määritelmä |
|-------|-----------|
| **Demo** | Esitys, jossa näytät projektia toiminnassa. Lyhyt, selkeä, näyttää arvoa. |
| **Dokumentaatio** (assessment) | Kirjallinen selitys projektista, jota voi lukea ilman sinun apua. README, ARCHITECTURE, SAFETY, TESTING. |
| **Rubric** (arviointiperuste) | Taulukko, joka näyttää, mitä eri arvosanat tarkoittavat. |
| **Arviointikriteeri** (criterion) | Yksi asia, jonka perusteella projektia arvioidaan (esim. "toiminnallisuus" tai "turvallisuus"). |
| **Edge case** | Epätavallinen tilanne, jonka olet testannut. |
| **Reflektio** | Ajattelu siitä, mitä oprit ja mitä tekisit toisin. |
| **Live-esitys** (live demo) | Demoa, jossa näytät agentin/botin toimivana, ei kuvakaappauksia. |
| **Narratiivi** | Tarina, jota kerrot demossa: mikä → miksi → miten → tulokset. |
| **Kriittinen ajattelu** | Kyky ajatella, miksi valitsit tämän lähestymistavan ja mitä voisit parantaa. |
